# reason-recall

"기억해 내라(회상)"가 아니라 "이 중에 있었나(재인)"로 묻는다. 매수일 전후 창의 공시로
후보를 만들고, 공시가 없어도 제시하는 사유 라이브러리와 직접 입력을 함께 객관식으로 낸다.
선택 결과는 클레임으로 변환해 원장에 기록한다 (이후 thesis-audit이 이 원장으로 판정).

## 컨트랙트 명령

```
python -m tools.run_skill recall --case <case_id> --choose r1,lib_theme --dry-run
python -m tools.run_skill recall --case <case_id> --choose r1,lib_theme --commit
python -m tools.run_skill recall --case <case_id> --custom-text "배당 보고 샀어요" --dry-run
```

- `--choose`: 후보 ID 목록(공시 `r1`.., 라이브러리 `lib_<type>`). 대화형(Codex)과 **같은 함수**를 호출.
- `--dry-run`: 원장 미기록(선택 확인 브리핑만). `--commit`: 스키마 검증 통과 시에만 기록.
- `--custom-text`: 직접 입력. 정적 키워드로 **보수적** 분류(확신 시에만 유형, 아니면 확인불가).

## 질문 세트 (PRD 4.2)

- **Q1 경로**: 유튜브·SNS / 커뮤니티·지인 / 뉴스 / 원래 알던 회사 / 직접 입력 (맥락 기록용, 판정 아님)
- **Q2 이유(핵심)**: 공시 후보 ≤4 (매수일 근접순, 각 rcp_no 보유) + 사유 라이브러리 + 직접 입력. **총 8개 이하**
- **Q3 기대(선택)**: 단기 / 수개월 / 장기 / 생각 안 해봄

## 결정성·안전 규약

- **후보는 스크립트 확정 분류분만** 사용 (에이전트 보조 분류는 후보 생성에 개입 안 함) → 후보 ID 재현.
- **미래 정보 차단**: 후보 생성 `as_of` = 매수일 + D+3. 그 이후 제출 공시는 후보에 없다.
- **악재성 제외**: 유상증자·CB 등은 후보에서 뺀다(그걸 보고 사는 초보자는 상정 안 함).
- **라이브러리 노출**: theme·price_anchor·unknown 항상, dividend는 배당 이력 있을 때, earnings("우량 기업")는 공시 후보에 실적이 없고 **영업이익 계정이 유효할 때만**(금융지주 안전핀 — RULE-EARN 오작동 방지).
- **골든 원장 배제**: 데모 골든 원장에는 custom을 넣지 않는다(LLM 분류 비결정성). custom은 정적 단위 테스트로만 검증(스냅샷 밖), 라이브 장면에서 즉석 시연.
- **직접 입력**: 원문을 사용자 발화로 보존, 분류는 1회 고정(thesis-audit이 재분류 금지), 저장 전 사용자 확인.

## 규약

- 파이프라인 로직은 순수 함수, CLI는 얇은 래퍼. 클레임/문구는 `render.py`·`textstore.py`.
- 원장 기록은 `ledger_store`(스키마 검증 통과 시에만 커밋)를 단일 진입점으로 거친다.
