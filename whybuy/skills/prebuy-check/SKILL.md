# prebuy-check

초보자 여정의 **첫 칸**(사기 전)을 지키는 스킬. 종목 이름 하나만 받아 그 회사의 공식 기록을
정해진 기준으로 스캔한다. **존재의 정직** — 짚이는 사실이 있으면 눈에 띄게 보여주고, 없으면
없다고 말하고 확인 가능한 사실을 판정 없이 제시한다. (부재의 정직의 반대편.)

## 컨트랙트 명령

```
python -m tools.run_skill prebuy --ticker <종목코드> [--as-of <YYYY-MM-DD>]
python tools/compliance_gate.py reports/prebuy/<종목코드>-<as_of>.md
```

- 입력은 종목코드 하나. 매수일·이유 불필요(아직 안 샀으니까).
- 기본 `as_of` = KRX 명단 스냅샷 기준일. 브리핑 상단에 "이 점검은 {as_of} 거래소 명단 기준"을 명시한다(KRX 명단은 시점 스냅샷이라 필수).

## 스캔하는 것 — 3층 (판정 아니라 사실)

- **층 1 거래소 조치 (KRX 명단)**: 관리종목·매매거래정지·투자주의/경고/위험·투자주의환기·불성실공시. `krx_client`(ticker 기반, corp 픽스처 무관).
- **층 2 재무 건전성 (DART)**: 자본잠식(**별도재무제표 OFS 기준** — 개별 법인)·연속 영업손실(config)·감사의견 비적정·계속기업 불확실성.
- **층 3 지배구조·주주가치 (DART)**: 횡령·배임·최대주주 2회↑ 변경·잦은 유상증자·CB(config)·영업정지.

기준값(연속손실·희석 건수 등)은 `config.yaml`에서 읽고 판정 시 브리핑에 함께 표기한다.

## 지키는 선

- **우리가 위험을 판단하지 않는다.** 거래소·회사·회계법인이 공식으로 남긴 사실만 전달. 각 항목에 "이는 공식 기록의 사실이며 위험 여부 판단은 제공하지 않습니다"를 명시하고, 묶음 제목도 "위험 신호"가 아니라 **"주의 깊게 볼 만한 공식 기록"**.
- **가격 격리**: 시가총액·주가·수익률로 위험을 판정하지 않는다(가격은 격리된 데이터, 규제선). 시총 순위는 전체 종목 가격이 필요해 제시하지 않는다 — 확인 가능한 사실은 공시 기반(감사의견·연속 흑자·배당·업력)만.
- **층위별 근거 정직**: DART 신호는 rcp_no 원문 링크, KRX 신호는 명단 출처+기준일 + "거래소 지정 조치는 개별 공시 원문이 제공되지 않아 명단 출처로 표기". 근거의 층위 차이를 숨기지 않는다.
- 브리핑은 저장 전 컴플라이언스 게이트 통과.

## 규약

- 파이프라인은 순수 함수(`build_prebuy`), CLI는 얇은 래퍼. 스캔 로직(`prebuy_check.py`)은 시세 임포트 0(가격 격리 테스트).
- 위험 목록 15종·KRX 픽스처·카카오 브리핑 스냅샷은 결정 주권 항목(아엔 승인).
