# G1 — 픽스처 수집 + 케이스 리허설 Implementation Plan

> **For agentic workers:** 03-DEV-PLAN의 G1 게이트. 게이트 루프 완주 필요. **골든 파일 게이트** — 픽스처·cases.json·REHEARSAL.md는 아엔 승인(부록 C) 후에만 커밋한다.

**Goal:** 실존 5종목(삼성전자·카카오·배당후보 3종)의 OpenDART/KRX 실데이터를 `whybuy/data/fixtures/`에 수집하고, 케이스 리허설로 각 케이스의 `buy_date`/`as_of`와 case-003 종목을 확정해 REHEARSAL.md를 낸다.

**Architecture:** `fetch_dart.py`는 순수 변환 함수(TDD 대상)와 얇은 네트워크 I/O 계층(통합 확인)으로 분리한다. 순수 함수: corp_cls 번역, 페이지네이션 조립, 유형별 분리, 날짜/정정 파싱, 재무 연결(CFS) 선택, 픽스처 직렬화. I/O: OpenDART HTTP + pykrx.

**환경 사실 (2026-07-09 확인):** 이 환경 = 아엔 로컬 (네트워크 가용). OpenDART 도달·키 유효(status 000, `whybuy/.env`의 `DART_API_KEY`). pykrx **종목 시세 OK**, **지수 조회 실패**(대안 경로 필요 — Task 8). 키는 `.env`에서만 읽고 로그·커밋에 노출 금지(crtfc_key 마스킹).

**Tech Stack:** Python 3.12(venv), requests, pykrx(fetch 전용), 표준 라이브러리(zipfile/xml/csv/json).
