# G10 — 매수 전 점검 (pre-buy check) Implementation Plan

> **For agentic workers:** 기존 G0~G9를 헤집지 않고 위에 얹는 독립 게이트. 게이트 루프 완주.
> **골든 파일 게이트** — 위험 신호 목록·KRX 픽스처·매수 전 점검 브리핑 스냅샷은 아엔 승인 후에만 커밋.

**Goal:** 투자자가 **아직 안 산** 종목 이름만 넣으면, 그 회사의 **공식 기록에 위험 신호로 정해진 사실**이 있는지 스캔해 두 갈래(있음/없음)로 정직하게 보여준다. 우리가 위험하다고 판단하지 않는다 — 거래소·회사·회계법인이 공식으로 남긴 사실만 전달한다.

**존재의 정직(원칙 5):** 지금까지 지켜온 **부재의 정직**(없으면 없다)의 반대편. 위험 신호가 **있으면 눈에 띄게 있다고** 말한다. 없으면 없다고 말하고 확인 가능한 사실을 판정 없이 제시한다. 둘 다 정직.

**제품 서사 확장:** 초보자 여정이 **매수 전 점검 → 매수 → 보유 중 → 매도 고민**이 된다. 지금까지 두 번째 칸부터 도왔는데, 이제 첫 칸부터 지킨다. "산 뒤만 보던 도구"에서 "사기 전까지 지키는 도구"로.

## KRX 오픈API 명세 대조 결과 (2026-07-10, 착수 전 선확인)

DART A~D 교훈대로 실명세를 계획과 대조함:

- **openapi.krx.co.kr 오픈API(발급 키 `KRX_API_KEY`)에는 층 1 위험 신호가 없다.** 카탈로그 7종(지수·주식·증권상품·채권·파생·일반상품·ESG)은 전부 시세·지수·상품 매매정보. 관리종목·매매거래정지·투자경고·불성실공시·실질심사·상장폐지는 **API 목록에 없음.** 공공데이터포털 KRX상장종목정보(별도 키)도 위험 플래그 없음(코드·시장구분·종목명·법인명만).
- **층 1 위험 신호는 `data.krx.co.kr` 정보데이터시스템에 있다** — 관리종목 현황/지정내역(개별), 매매거래정지 내역(개별), 투자주의·경고·위험종목, 불성실공시법인. 개별종목 조회·CSV 다운로드 가능. **G1 지수 CSV와 동일한 웹 통계 시스템**(키 불필요).
- **결정(아엔, 2026-07-10):** 층 1 = **data.krx.co.kr 수동 CSV**. 아엔이 `data/fixtures/raw/`에 제공(EUC-KR): `krx_admin_issues`(관리종목)·`krx_trading_halt`(매매거래정지)·`krx_investment_alert`(투자주의)·`krx_investment_warning`(투자경고)·`krx_warning_kosdaq`(투자주의환기 코)·`krx_unfaithful_disclosure`(불성실공시). 투자위험종목 0건(빈 카테고리). 실질심사·상폐사유 CSV 없음(데이터 없음 처리). `KRX_API_KEY`는 시세용이라 위험 신호에 **미사용**하되 `.env` 마스킹 보존.

## 위험 신호 목록 (승인 대상 D1)

각 신호: 정의 · 판정 기준(코드로 고정) · 소스 · 근거 원문.

> **표현 규칙(아엔, 규제 선):** 내부 코드·설계는 "위험 신호 15종"으로 관리하되, **사용자 브리핑에는 "위험 신호"라는 단어를 앞세우지 않는다.** 각 항목을 그 사실의 **공식 명칭 그대로** 부르고 **"공식 기록의 사실이며 위험 여부 판단은 제공하지 않습니다"**를 명시. 묶음 제목도 "위험 신호"가 아니라 **"주의 깊게 볼 만한 공식 기록"**. (근거: 층 2·3은 회계법인 선언·회계 정의·회사 신고 — 제3자의 사실이지 우리 판단 아님. 그래서 선 안쪽. 표현 상수는 textstore.)
> 예) ❌ "위험 신호: 자본잠식" → ✅ "자본총계가 자본금보다 적습니다(자본잠식) [원문]. 공식 기록의 사실이며 위험 여부 판단은 제공하지 않습니다."

**층 1 (KRX, data.krx.co.kr CSV — 아엔 제공):**
1. 관리종목 지정 — `krx_admin_issues` 목록에 종목코드 존재
2. 투자주의환기종목(코) — `krx_warning_kosdaq` 존재
3. 상장적격성 실질심사 대상 — **별도 CSV 없음 → 데이터 없음(정직 비활성화)**. 관리종목·매매거래정지 사유 텍스트로 간접 노출 여부만 Task 1에서 확인
4. 상장폐지 사유 발생 — **별도 CSV 없음 → 데이터 없음(정직 비활성화)**
5. 매매거래정지 — `krx_trading_halt` (정지일·정지사유)
6. 불성실공시법인 지정 — `krx_unfaithful_disclosure` (지정일·벌점·불성실유형)
7. 투자주의/투자경고/투자위험 — `krx_investment_alert`(주의)·`krx_investment_warning`(경고)·**투자위험 0건(빈 카테고리)**

**층 2 (DART):**
8. 감사의견 비적정 — report_items `audit_opinion.adt_opinion` ∈ {의견거절, 부적정, 한정}
9. 자본잠식 — financials 자본총계 < 자본금 (부분잠식) / 자본총계 ≤ 0 (완전잠식)
10. 계속기업 불확실성 — 감사보고서 강조사항/특기사항 "계속기업" 문구. **⚠ 픽스처에 이 텍스트 실재 여부를 Task 3에서 먼저 확인 — 없으면 이 신호만 데이터 없음으로 정직 비활성화, 나머지 14종 진행(억지로 만들지 않음).**
11. 연속 영업손실 — financials 영업이익 **4분기(1년) 연속 음수** (config `consecutive_op_loss_quarters: 4` — 2분기는 정상 회사도 흔해 오탐)

**층 3 (DART):**
12. 횡령·배임 발생 — 공시 유형 litigation ∧ 제목 "횡령|배임"
13. 최대주주 2회 이상 변경 — disclosures "최대주주변경" 건수 ≥ 2 (G8에서 KT&G 실증)
14. 잦은 유상증자/CB/BW — capital_change(증자·CB·신주인수권) **최근 1년 내 ≥ 3건** (config `dilution_window_days: 365`, `dilution_count_threshold: 3`)
15. 영업정지/주된 사업 중단 — business_halt

## 지켜야 할 선 (가격 격리·규제 선 유지)

- **재무 지표에 좋다/나쁘다 선을 긋지 않는다.** 사실만 나열하고 "판정이 아니다"를 명시. 단, 층 2·3의 위험 신호는 **거래소·회계법인이 공식 지정한 사실**(예: 자본잠식은 회계 정의)이라 "신호 존재" 자체가 사실 — 우리의 가치 판단이 아님.
- **시가총액 순위는 사실이라 표기 가능**하나, "얼마 이상이면 우량"이라는 **기준은 우리가 정하지 않는다.**
- **시가총액·주가·수익률로 위험 여부를 판정 금지.** 그건 격리한 가격 데이터이고 규제 선(가치 평가)에 닿는다. prebuy-check는 verdict_engine처럼 **시세를 위험 판정에 쓰지 않는다**(가격 격리 유지 — 소스 스캔 테스트).
- 위험 있음 브리핑: "이건 사지 말라는 판단이 아니라 공식 기록에 있는 사실"임을 고정 문구로 명시(textstore).
- 없음 브리핑: "공식 기록상 위험 신호 없음" + 확인 가능한 사실을 **판정 없이** 제시 — 시가총액 순위, 감사의견, 연속 흑자 여부, 배당 이력, 희석 이벤트 유무 정도.

## 동작 흐름

1. 종목 이름만 입력 (매수일·이유 불필요 — 아직 안 샀으니까). corp_code/ticker 해소.
2. 층 1(KRX) + 층 2·3(DART)을 위험 신호 기준으로 스캔.
3. 결과 두 갈래:
   - **위험 신호 있음** → 각 항목을 원문 링크와 함께 나열 + "판단 아니라 공식 기록의 사실" 명시.
   - **위험 신호 없음** → "공식 기록상 위험 신호 없음" + 확인 가능한 사실(판정 없이).

**데모 = 카카오.** 위험 종목을 억지로 찾지 않는다. 카카오 넣어 나오면 나오는 대로, 안 나오면 안 나오는 대로 정직하게. (기능의 존재를 보여주는 것.) 검출 능력 증명은 **별도 테스트 픽스처**(실제 관리종목 1종의 KRX CSV)로 — 데모 아님.
