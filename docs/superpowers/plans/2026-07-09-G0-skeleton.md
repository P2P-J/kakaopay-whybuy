# G0 — 저장소 골격과 환경 Implementation Plan

> **For agentic workers:** 이 계획은 03-DEV-PLAN의 G0 게이트 하나를 실행 가능한 체크리스트로 펼친 것이다. 게이트 단위 루프(구현→자체검토→테스트→완료판정→회귀→로그→커밋→보고)를 완주해야 G0가 끝난다.

**Goal:** PRD §8 구조대로 `whybuy/` 플러그인 골격·스켈레톤·환경(venv/의존성)을 세우고, G0 완료판정 3종을 exit 0으로 통과시킨다.

**Architecture:** 플러그인은 저장소 루트의 `whybuy/` 하위 디렉토리에 자립적으로 둔다(기존 로그 하네스와 분리 — 아엔 확정). 파이썬 모듈은 이 게이트에서 스켈레톤(도크스트링 + 후속 게이트 TODO)만 만들고, 실로직은 각 담당 게이트에서 채운다. `config.yaml`·`tools/textstore.py`는 이 게이트에서 실내용을 채우는 두 예외다(0.7-1·0.7-2 seam).

**Tech Stack:** Python 3.12, uv(venv·의존성 관리 — pip 부재로 채택), requests·jsonschema·pyyaml·pytest·mcp(SDK, 버전고정).

**환경 사실 (2026-07-09 개발환경 확인):** 시스템 python3.12에 requests 2.31.0/jsonschema 4.10.3/pyyaml 6.0.1 존재. `python`(bare)·pytest·mcp·pip 부재. `uv`/`uvx` 존재 + PyPI 접근 가능 → uv로 whybuy/.venv 구성해 `python`/`pytest`를 확보한다.
