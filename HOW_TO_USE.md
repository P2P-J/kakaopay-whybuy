# HOW TO USE — 왜샀지 직접 돌려보기

이 문서는 **실제로 플러그인을 돌려보며 동작을 두 눈으로 확인**하는 실행 안내서다.
각 명령 아래에 **실제 출력**을 함께 실었으니, 돌렸을 때 화면과 대조하면 된다.

> **폴더 이름**: 제출 ZIP에서는 플러그인이 `src/`, 이 저장소에서는 `whybuy/` 안에 있다.
> 아래는 `src/` 기준이며, 저장소에서 볼 때는 `cd src` 를 `cd whybuy` 로만 바꾸면 나머지는 동일하다.
> 출력의 `저장:` 경로도 `src/` ↔ `whybuy/` 만 다르고 **브리핑 내용은 완전히 같다.**

---

## 0. 준비 — 키 없이 픽스처 모드로 돈다

```bash
cd src
python -m venv .venv && . .venv/bin/activate     # 또는: uv venv .venv
pip install -r requirements.txt                   # 또는: uv pip install -r requirements.txt
```

`WHYBUY_MODE`는 기본이 `fixture`다. **API 키도, 인터넷도 필요 없다.** 아래 모든 명령이
저장소에 함께 담긴 오프라인 픽스처만으로 동작한다.

## 1. 매수 전 점검 (prebuy-check) — "사기 전에 알아둘 게 있나?"

### (a) 위험 신호가 **없는** 종목 — 카카오

```bash
.venv/bin/python -m tools.run_skill prebuy --ticker 035720
```

터미널:
```
저장: reports/prebuy/035720-2026-07-10.md · 주의 항목 없음
게이트 PASS
```

산출물(`reports/prebuy/035720-2026-07-10.md`):
```markdown
# (주)카카오(035720) 매수 전 점검

> 이 브리핑은 2026-07-10까지 제출된 공시 기준입니다.
> 이 점검은 2026-07-10 거래소 명단 기준입니다.

## 주의 깊게 볼 만한 공식 기록
- 공식 기록상 주의 항목: 발견되지 않았습니다. (2026-07-10 거래소 명단 기준)

## 참고로 확인 가능한 사실
- 최근 감사의견: 적정의견 [원문](https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20260318001423)
- 최근 8분기 연속 영업흑자
- 배당: 주당 현금배당금 75원 [원문](https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20260318001423)
- 업력: 1995년 설립

> 이는 공식 기록의 사실 정리이며, 위험 여부나 투자 적합성에 대한 판단은 제공하지 않습니다.
```

**없으면 "없다"고 말한다(부재의 정직).** 그러면서 감사의견·연속흑자·배당·업력 같은
**확인 가능한 사실**은 그대로 보여준다(존재의 정직). 위험 여부 판단은 하지 않는다.

### (b) 위험 신호가 **있는** 종목 — 오에스피(거래소 명단), 케이티앤지(DART 지배구조)

```bash
.venv/bin/python -m tools.run_skill prebuy --ticker 368970    # 오에스피 — 거래소 층1
.venv/bin/python -m tools.run_skill prebuy --ticker 033780    # 케이티앤지 — DART 층3
```

오에스피 산출물 발췌:
```markdown
## 주의 깊게 볼 만한 공식 기록
- 관리종목: 시가총액 미달 · 출처: 한국거래소 관리종목 지정 현황, 2026-07-10 기준 [KRX](https://data.krx.co.kr/)
- 불성실공시법인 지정: 자기주식 취득 신고주식수 미만 매매거래주문 · 출처: 한국거래소 불성실공시법인 지정 현황, 2026-07-10 기준 [KRX](https://data.krx.co.kr/)
> 거래소 지정 조치는 개별 공시 원문이 제공되지 않아 명단 출처로 표기합니다.
> 재무·지배구조(DART 공시) 항목은 이 데모 픽스처 범위 밖이라 확인하지 않았습니다 — 라이브 모드에서 조회됩니다.
```

케이티앤지 산출물 발췌:
```markdown
## 주의 깊게 볼 만한 공식 기록
- 최대주주 변경 4회: 기준: 최대주주 변경 2회 이상 [원문](https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20260122800445)
- 영업정지·사업중단 관련 공시: 영업정지(종속회사의주요경영사항) [원문](https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20260512800697)
```

**사기 전에, 위험이 이미 공식 기록에 적혀 있으면 그 사실을 그대로 보여준다.** 거래소
명단(층1)은 명단 출처로, DART 항목(층2·3)은 공시 원문 링크로 근거를 연다. 확인하지
않은 범위는 "확인하지 않았다"고 정직하게 밝힌다.

📸 **캡처 지점 — 매수 전 점검**: (a) 카카오 "없음" 화면과 (b) 오에스피/케이티앤지
"있음" 화면을 나란히.

## 2. 매수일 타임라인 (buy-timeline) — "산 뒤로 뭐가 달라졌어?"

```bash
.venv/bin/python -m tools.run_skill timeline --case case-001    # 삼성전자
```

터미널: `저장: reports/case-001/timeline-2025-04-09.md` / `게이트 PASS`

산출물(`reports/case-001/timeline-2025-04-09.md`):
```markdown
# 2025-03-25, 당신이 산 이후 삼성전자(주)에 일어난 일

> 이 브리핑은 2025-04-09까지 제출된 공시 기준입니다. (매수 후 15일 경과)
> 보유 수익률(참고): -11.4% (기준: 실제 체결가 59800원, 배당 미반영) · 같은 기간 코스피 -12.3% → 시장 대비 유사

## 변한 것
- 2025-04-08: 실적(잠정 또는 정기)이 공시되었습니다 [연결재무제표기준영업(잠정)실적(공정공시), 원문](https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20250408800003) · 최근 확정 영업이익 32.7조원, 전년 동기 대비 +398% (20244Q·CFS)
> 그 외 공시 3건은 접었습니다 (임원 지분변동·기업설명회 등 반복성 공시).

## 변하지 않은 것
- 최대주주: 삼성생명보험㈜ (8.51%) (매수 시점과 동일) [근거 공시 원문](https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20250311001085)
- 감사의견: 적정의견 (최근 사업보고서 기준) [근거 공시 원문](...)
- 배당 정책: 주당 현금배당금 1,446원 (매수 시점과 동일) [근거 공시 원문](...)

## 주가가 크게 움직인 날 (참고)
- 2025-03-31: -4.0% ← 해당 기간 공시 없음. 회사의 공식 기록상 변한 사실은 없습니다 (시장 전체 흐름이나 수급 요인일 수 있으며, 이는 공시로 확인되지 않는 영역입니다).
- 2025-04-07: -5.2% ← 매칭: 연결재무제표기준영업(잠정)실적(공정공시)
```

**매수일(2025-03-25)을 원점으로, 그 뒤에 실제로 달라진 것만 재정렬한다.** 수익률은
시장 대비로 병기하고(판정 아님, 참고), 급변일에 매칭 공시가 없으면 인과를 지어내지 않고
"변한 사실 없음"이라고 못 박는다. 2025-04-07 −5.2% 급락일에 **좋은 실적**이 매칭된 것도
그대로 병치한다 — 설명을 만들지 않는다.

📸 **캡처 지점 — 타임라인**: 이 "변한 것 / 변하지 않은 것 / 주가가 크게 움직인 날" 화면.

## 3. 왜 샀는지 재구성 (reason-recall) — "나 이거 왜 샀더라?"

```bash
.venv/bin/python -m tools.run_skill recall --case case-001 --choose r1 --dry-run
```

출력:
```markdown
# 삼성전자(주) — 산 이유 재구성 (확인)

> 이 브리핑은 2025-03-28까지 제출된 공시 기준입니다. (후보는 매수일 전후 창의 공시로만 생성)

## 기록할 논거
- [공시 근거] 실적 개선 기대: “전사 영업이익이 개선 추세다” (원문 rcp 20250311001085)

[dry-run(미기록)] 논거 1건
```

산 이유를 서술형으로 캐묻지 않는다. **매수일 주변 공시로 만든 객관식 후보**를 제시하고,
고른 논거(`--choose r1`)를 원장에 기록한다(`--dry-run`은 기록 없이 미리보기). 이 논거가
다음 단계(판정)의 대상이 된다.

📸 **캡처 지점 — 논거 재구성**: 객관식 논거 후보가 뜨는 이 화면.

## 4. 논거 판정과 거울 (thesis-audit --mirror) — "그 이유 아직 유효해?"

### (a) 반증 — 카카오 (기록한 이유가 공시로 뒤집힌 경우)

```bash
.venv/bin/python -m tools.run_skill audit --case case-002 --mirror
```

산출물(`reports/case-002/mirror-2025-05-14.md`):
```markdown
# 카카오 — 매도 버튼 앞에서

> 이 브리핑은 2025-05-14까지 제출된 공시 기준입니다. (매수 후 79일 경과)

## 그때 — 당신이 기록한 매수 이유
- 실적 개선 기대: “전사 영업이익이 개선 추세다”

## 지금 — 2025-05-14
> 보유 수익률(참고): -4.0% · 같은 기간 코스피 -0.2% → 시장 대비 하회
- 실적 개선 기대: **반증** — 영업이익 20251Q, -12% YoY (감익) [원문](https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20250514001239) (rule: RULE-EARN-01)

### 당신이 고르지 않았지만 알아야 할 새로운 사실
- 2025-03-11: 주요사항보고서(감자결정) [원문](https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20250311001242) — 감자: 자본을 줄이는 결정으로, 주식 수·자본 구조에 영향을 줄 수 있습니다

## 지금 팔고 싶은 게 하락한 가격 때문인가요, 회사가 달라져서인가요?

반증은 회사에 대한 평가가 아니라 사용자의 논거 대비 공시 사실의 판정입니다.
```

### (b) 유효 — 삼성 (가격은 빠졌지만 논거는 살아 있는 경우)

```bash
.venv/bin/python -m tools.run_skill audit --case case-001 --mirror
```

산출물 발췌(`reports/case-001/mirror-2025-04-09.md`):
```markdown
## 지금 — 2025-04-09
> 보유 수익률(참고): -11.4% · 같은 기간 코스피 -12.3% → 시장 대비 유사
- 실적 개선 기대: **유효** — 영업이익 20244Q, +398% YoY (개선) [원문](https://dart.fss.or.kr/dsaf001/main.do?rcpNo=20250311001085) (rule: RULE-EARN-01)

### 당신이 고르지 않았지만 알아야 할 새로운 사실
- 논거와 무관한 새로운 중대 사실은 없었습니다.

## 가격은 빠졌지만 회사가 달라졌다는 공시는 아직 없습니다. 지금 팔려는 마음은 사실에서 오나요, 가격에서 오나요?
```

**팔기 직전, 그때의 이유가 지금도 유효한지 공시 원문으로 판정한다.** 판정마다 규칙 번호
(`RULE-EARN-01`)와 원문 링크가 붙어 근거를 연다. 두 케이스를 나란히 보면 힘이 드러난다 —
카카오는 손실 + **반증**(가격도 사실도 나빠짐), 삼성은 손실 + **유효**(가격은 빠졌지만
사실은 그대로). 거울의 마지막 질문은 판단을 내려주지 않고, **가격 때문인지 사실 때문인지**
사용자에게 되돌려준다.

📸 **캡처 지점 — 거울 브리핑 & DART 원문**: (a) 카카오 반증 화면(킬러) + (b) 삼성 유효
화면을 병치. 판정줄의 `[원문](https://dart.fss.or.kr/...)` 링크는 실제 DART 공시로
열린다 — 링크가 살아 있음은 `.venv/bin/python tools/link_check.py`로 재검증된다.
